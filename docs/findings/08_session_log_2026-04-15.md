## Summary of Work Completed

### 1. PDF Extraction & Chunking (Earlier in Session)
- Used **pypdf** to extract text from 6 wildfire mitigation PDFs in `docs/data_docs/`
- Suppressed noisy pypdf warnings with `logging.getLogger("pypdf").setLevel(logging.ERROR)` and `warnings.filterwarnings`
- Configured sliding-window chunking: **350 words, 70-word overlap, 25-word minimum**
- Produced **510 chunks** saved to `results/all_chunks.json`
- Attempted pdfminer.six as an alternative but it produced garbled output — reverted to pypdf

### 2. Embedding Pipeline & Grid Search
- Built embedding helper with robust error handling:
  - `MAX_EMBED_CHARS = 2000` truncation for oversized chunks
  - `_embed_single_safe()` fallback when batch `/api/embed` returns HTTP 400
  - Zero-vector fallback for unembeddable texts
  - Batch-to-single fallback on HTTP errors
- Ran grid search across 5 Ollama embedding models (2 of 5 succeeded due to model availability)
- **Winner:** `nomic-embed-text:latest` — dim=768, MRR=0.984 on evaluation set
- Saved: `results/best_embeddings.npy`, `results/embedding_grid_search.csv`, `results/rag_config.json`

### 3. ChromaDB Vector Store
- Built persistent ChromaDB collection `wildfire_docs` in `vector_store/`
- Indexed all 510 chunks with their embeddings
- Verified retrieval quality with test queries

### 4. Neo4j Knowledge Graph Setup
- **Problem:** Original Neo4j instance on Viridian cloud server was unreachable (WinError 10061)
- **Solution:** Switched to local Docker container managed via Python `docker` SDK (v7.1.0)
  - Container name: `ember-neo4j`
  - Image: `neo4j:5`
  - Ports: 7687 (Bolt), 7474 (HTTP)
  - Auth: `neo4j/password`
  - Functions: `_ensure_neo4j_container()` + `_wait_for_neo4j()` for lifecycle management

### 5. Knowledge Graph Schema Design
Built a multi-layer knowledge graph with the following structure:

**Core Layer (from RAG corpus):**
- 6 **Document** nodes (one per source PDF)
- 510 **Chunk** nodes with 768-dim embedding vectors
- 26 **Concept** nodes (domain vocabulary: Large Fire, FireSmart, SHAP, etc.)
- Relationships: `CONTAINS`, `MENTIONS`, `RELATED_TO`
- Vector index: `ember_chunk_vectors` (cosine similarity, dim=768)

**Analytical Knowledge Layer (from ML findings — added tonight):**
- 12 **Feature** nodes with importance_rank, gini_importance, shap_direction
- 4 **RiskLevel** nodes with probability thresholds (LOW / MODERATE / HIGH / VERY HIGH)
- 10 **MitigationAction** nodes (5 operational + 5 preventive)
- 6 **DataInsight** nodes citing key findings from the analysis
- New relationships: `PREDICTS`, `TRIGGERS_ACTION`, `ASSESSES`, `RECOMMENDS`
- All analytical nodes carry dual `:Concept` label so existing GRAG tools discover them automatically

## Key Technical Decisions

| Decision | Rationale |
|---|---|
| pypdf over pdfminer.six | pdfminer produced garbled output on these PDFs |
| nomic-embed-text:latest | Won grid search with MRR=0.984 (dim=768) |
| Docker-managed Neo4j | Viridian cloud instance unreachable; Docker gives local control |
| Dual `:Concept` labels on analytical nodes | Lets existing GRAG tools discover new structure without code changes |
| 350-word chunks, 70-word overlap | Balanced relevance and context window coverage |
| MAX_EMBED_CHARS=2000 | Prevents HTTP 400 from Ollama on oversized texts |
